# Coverage Jira Tasks Skill

Generate and create Jira tasks for code coverage onboarding from an audit CSV.

## Overview

Two-phase workflow:

1. **Dry run** — reads audit CSV, classifies repos, generates one `.md` file per task locally
2. **Create tasks** — reads generated files, creates Jira issues, links to epic

## Prerequisites

- Audit CSV from the `coverage-audit` skill (or manually created)
- Jira API token: https://id.atlassian.com/manage-profile/security/api-tokens
- `certifi` Python package (for macOS SSL): `pip install certifi`

## Usage

### Phase 1: Generate task files

```bash
python scripts/dry-run.py \
  --csv <org>-audit.csv \
  --org <github-org> \
  --output-dir ./jira-tasks-draft/
```

Filter to specific repos:
```bash
python scripts/dry-run.py \
  --csv <org>-audit.csv \
  --org <github-org> \
  --output-dir ./jira-tasks-draft/ \
  --repos quay-operator,mirror-registry
```

Filter by task type:
```bash
python scripts/dry-run.py \
  --csv <org>-audit.csv \
  --org <github-org> \
  --output-dir ./jira-tasks-draft/ \
  --types onboard-unit,fix-codecov
```

Both `--repos` and `--types` can be combined.

Review generated files in `./jira-tasks-draft/`. Delete or edit as needed.

### Phase 2: Create in Jira

```bash
export JIRA_USERNAME="user@redhat.com"
export JIRA_API_TOKEN="your-token"

python scripts/create-tasks.py \
  --input-dir ./jira-tasks-draft/ \
  --epic <EPIC-KEY> \
  --project <PROJECT-KEY>
```

For non-default Jira instances:
```bash
export JIRA_URL="https://your-instance.atlassian.net"
# or pass --jira-url flag
```

Markdown in task files is automatically converted to Jira wiki markup before upload.

## Task Types and Labels

Every task gets the `codecov-onboarding` label plus a type-specific label:

| Task Type | Label | Priority | Condition |
|-----------|-------|----------|-----------|
| Fix Codecov config | `fix-codecov` | Critical | Has Codecov but missing flags or config-only |
| Verify Codecov setup | `verify-codecov` | Major | Has Codecov with flags, needs confirmation |
| Onboard unit tests | `unit-tests` | Normal | Has tests + CI, no Codecov |
| Set up CI first | `needs-ci` | Normal | Has tests but no CI detected |
| Add tests first | `needs-tests` | Minor | Has CI but no tests, or placeholder tests |
| Investigate + onboard | `investigate` | Minor | Has language but unknown tests and no CI |
| Onboard e2e via Coverport | `e2e-tests` | Minor | Has e2e tests in Go/Python/JS/TS |

High-star repos (100+) get priority boosted automatically.

## Onboard Column (Recommended)

If the CSV was generated by the `coverage-audit` skill, it includes an `Onboard` column
with `TRUE`/`FALSE` values. The user curates these in Google Sheets (checkboxes), then
downloads as CSV.

When `dry-run.py` detects the `Onboard` column, it automatically filters to `TRUE` rows —
no need to pass `--repos` or answer "which repos?" interactively.

The `--repos` flag overrides the Onboard column if you want to narrow further.

If the CSV has no `Onboard` column (manual CSV or older audit), the skill falls back to
interactive repo selection.

## Filtering

Repos are automatically skipped if:
- Category is not `application`
- Fork = `yes`
- Archived = `yes`
- No language AND no CI detected (config/docs repos)

## Task File Format

Each generated `.md` file has YAML frontmatter:

```yaml
---
summary: "repo-name: Onboard unit test coverage to Codecov"
priority: "Normal"
type: "Task"
labels: "codecov-onboarding, unit-tests"
---
```

Followed by a markdown description with: Objective (with star count and repo description), Current State table, Key contacts, language-specific Steps, AI skill reference, and Verification checklist.
